=== ANSWER ===

To get started with the Task Notes Microservice API, follow this quick start workflow:

1. **Create**: Send a POST request to `/notes` with a JSON payload containing the note details. 
   Example: `POST /notes` with `{"title": "New Note", "content": "This is a new note"}`

2. **List**: Send a GET request to `/notes`.
   Example: `GET /notes`

3. **Get**: Send a GET request to `/notes/{id}`, replacing `{id}` with the ID of the note you want to retrieve.
   Example: `GET /notes/123` [SOURCE 1]

4. **Update**: Send a PUT request to `/notes/{id}`, replacing `{id}` with the ID of the note you want to update.
   Example: `PUT /notes/123` with `{"title": "Updated Note", "content": "This note has been updated"}` [SOURCE 1]

5. **Delete**: Send a DELETE request to `/notes/{id}`, replacing `{id}` with the ID of the note you want to delete.
   Example: `DELETE /notes/123` [SOURCE 1]

Note that the API returns JSON for both success and error responses [SOURCE 2]. Also, the collection endpoint accepts both `/notes` and `/notes/` in runtime routing, but the canonical path in this spec is `/notes` [SOURCE 2].

1. **Create a note**: Send a `POST` request to `/notes` with a JSON body containing note details.

   - Expected response: `201` with a created note object.

2. **List notes**: Send a `GET` request to `/notes`. Optionally, filter by tag using `?tag=...`.
   - Example: `GET /notes` or `GET /notes?tag=personal`
   - Expected response: `200` with an array of note objects.

3. **Get a note**: Send a `GET` request to `/notes/{id}`.
   - Example: `GET /notes/123` (replace `123` with the actual note ID)
   - Expected response: `200` with a note object.

4. **Update a note**: Send a `PUT` request to `/notes/{id}` with a JSON body containing updated note details.
   - Example: `PUT /notes/123` with body `{"text": "Updated note", "tag": "work"}`
   - Expected response: `200` with an updated note object.

5. **Delete a note**: Send a `DELETE` request to `/notes/{id}`.
   - Example: `DELETE /notes/123`
   - Expected response: Typically `204` (No Content) or similar success indicator.
